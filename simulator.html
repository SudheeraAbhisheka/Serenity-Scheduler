Multi-Server Simulator

This system simulates multiple servers processing tasks in real-time and supports automatic detection of new servers as well as server crashes.

  • Heartbeat mechanism and task recovery
  • Identifying new servers automatically

Heartbeat Mechanism & Task Recovery

Each server sends periodic heartbeat signals. A "heartbeat checking thread" monitors these signals and detects any stopped or crashed server. On a crash:

  • Tasks currently in that server's queue are recovered.
  • Tasks "in transit" between the scheduler and the server are also recovered.

Identifying New Servers Automatically

When a new server is added, the scheduler is notified automatically, so that it can start distributing tasks to the new server without needing a manual refresh.
