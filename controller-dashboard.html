Controller & Dashboard

Controller

  • Creating new servers: Add servers one at a time or add multiple servers with the same speed/capacity.
  • Selecting the scheduling algorithm: Either "Complete and then fetch" or "Load balancing" can be used, with or without priority scheduling.
  • Generating tasks: Send messages via Kafka or RabbitMQ. Use either a simple bulk addition (by specifying a count) or send messages at fixed intervals. You can also increase the sending threads if sending a large number of tasks at once causes a bottleneck.

Dashboard

  • Real-time server load charts: Powered by WebSockets and React-eCharts (or another charting library).
  • Summary after tasks complete: Shows how many tasks each server handled and the total wait time of every task from generation to start of processing.
  • Crash a server: Simulate server malfunction for testing the heartbeat and recovery mechanism.
  • Task table view: A view of the database (Cassandra) entries in the order of process-completion time.
